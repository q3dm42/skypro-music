# Skypro Music

Учебный проект музыкального сервиса на Next.js и TypeScript.

## Что готово в HW-01

- Главная страница перенесена в TSX.
- Интерфейс разложен на компоненты в `src/components`.
- Для компонентов используются CSS Modules.
- Статические изображения и SVG-иконки находятся в `public`.
- Логотип и обложки подключены через `next/image`.
- Навигационные элементы подключены через `next/link`.
- Шрифт Montserrat подключен в корневом `layout.tsx`.

## Что готово в HW-02

- Бургер-меню открывается и закрывается через React state.
- Треки выводятся через `map` из типизированных моковых данных.
- Фильтры по исполнителю, году выпуска и жанру строятся динамически.
- Одновременно открывается только одно окно фильтра.
- Активный фильтр подсвечивается и показывает количество вариантов.
- Дубликаты в списках фильтров удаляются.

## Что готово в HW-03

- Треки загружаются из Skypro.Music API.
- Подключен Redux Toolkit и `react-redux`.
- В Redux хранится текущий трек и состояние воспроизведения.
- Клик по треку выбирает его, запускает аудио и показывает активную точку.
- Play/Pause управляет нативным `<audio>` через `useRef`.
- В плеере отображаются название и автор текущего трека.
- Точка у активного трека пульсирует только во время воспроизведения.

## Что готово в HW-04

- Текущий плейлист сохраняется в Redux при выборе трека.
- Работают переходы к следующему и предыдущему треку.
- После завершения трека автоматически включается следующий.
- Shuffle переключает случайный порядок воспроизведения.
- Loop зацикливает текущий трек и подсвечивает кнопку.
- Добавлен прогресс-бар с перемоткой.
- Отображается текущее и полное время трека.
- Слайдер громкости управляет уровнем звука.

## Что готово в HW-05

- Реализованы страницы входа и регистрации.
- Ошибки API отображаются пользователю в форме.
- После регистрации выполняется переход на страницу входа.
- После входа сохраняются пользователь и токены, затем выполняется переход на главную.
- Главная страница получает треки из API и передает их в фильтры.
- Страницы подборок `/selection/[id]` загружают подборку из API.
- Главная и подборки используют общий `AppShell`.
- Для страниц с API добавлен loader и пользовательское сообщение об ошибке.

## Что готово в HW-06

- Авторизованный пользователь может добавлять и удалять лайки у треков.
- Лайки работают в списке треков и в плеере.
- Состояние лайков и количество обновляются без перезагрузки страницы.
- Добавлена страница `/favorites` с избранными треками пользователя.
- Неавторизованный доступ к избранному перенаправляется на вход.
- Удаление лайка на странице избранного убирает трек из списка.
- Выход из аккаунта очищает данные сессии.
- Если выйти на странице избранного, выполняется переход на главную.
- Авторизованные запросы обновляют access token через refresh token при 401.

## Что готово в HW-07

- Реализован комбинированный поиск по названию трека.
- Реализованы фильтры по исполнителю и жанру.
- Реализована сортировка по дате выпуска.
- Поиск, фильтры и сортировка работают совместно.
- При пустом результате показывается сообщение.
- Чистые функции фильтрации и сортировки вынесены отдельно.
- Добавлены тесты для чистых функций и компонентов `Search`, `Filter`, `Centerblock`.
